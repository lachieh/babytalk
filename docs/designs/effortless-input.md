## Vision

### 10x Check

The 10x version makes the entire tracking experience disappear. You don't "track" anymore — you just live, and BabyTalk knows. Ambient awareness detects events. Predictive prompts anticipate needs. Partner handoff is automatic. Pediatrician exports are one voice command away. The app feels like a calm, competent co-parent who never sleeps, never forgets, and never judges.

**This plan delivers the foundation layer.** Ambient awareness, predictive prompts, and pediatrician exports are planned for Phase 2 (Context Engine).

### Platonic Ideal

The user feels *relief*. Not "oh, a baby tracking app" — but "oh thank god, someone actually understands what it's like." Every interaction is warm, contextual, and anticipatory. The design is so minimal it barely feels like software — a gentle pulse showing the baby's rhythm, a voice you can talk to, a partner who always knows what happened. No dashboards. No charts. No settings screens. Just a conversation with something that knows your baby.

## Critical Discovery: iOS Haptics Gap

`navigator.vibrate()` is not supported on iOS Safari. Resolution: progressive enhancement — haptics on Android, audio cues on iOS (soft chime for "logged," double-tone for "timer started"). Visual countdown indicator works everywhere as baseline. All feedback modalities have a visual equivalent for accessibility (deaf/hard-of-hearing users).

## Implementation Approach

The approach builds on existing code (voice-button.tsx, tambo/tools.ts, quick-actions.tsx) and adds:
- New: service worker, IndexedDB offline queue, web app manifest, push subscription API endpoint
- Enhanced: voice button component (hold-to-talk, continuous, silence detection, audio/haptic feedback)
- Enhanced: Tambo system prompt (smart defaults, warm confirmations, handoff summaries)
- New: status widget component, dark mode system, onboarding page redesign
- Enhanced: quick-actions component (reordering, swipe gestures, inline detail panels)

### Smart NLP Defaults — Concrete Examples

- "fed the baby" → defaults to breastfeeding, same side as last feed, current time as startedAt
- "she ate for 15 minutes" → breast, infer side from last feed, endedAt = now, startedAt = 15 min ago
- "diaper change" → wet=true, soiled=false (most common), current time
- "baby slept" → starts a sleep timer, location=last used location
- First-time user with no history → ask for method/side explicitly, then learn from patterns

### Silence Detection Implementation

Custom silence detection using AudioContext + AnalyserNode (not relying on Web Speech API's built-in timeouts, which vary across browsers). The AnalyserNode monitors audio amplitude; when it drops below a threshold for 2.5s (configurable), auto-submit triggers. This approach is consistent across Chrome, Safari, and Edge.

### Quick-Action Swipe Gestures

Swipe direction: **vertical only** (swipe down to log with defaults). Horizontal swipe is avoided because it conflicts with browser back/forward navigation and PWA gestures. Pull-to-refresh is disabled in standalone PWA mode via CSS `overscroll-behavior: none`.

### Offline Queue Details

- **Storage limit:** 100 pending mutations max. At 10 events/day, that's 10 days offline — more than enough.
- **Staleness:** Events older than 7 days in the queue are flagged to the user: "These events are old — still want to sync them?"
- **Conflict resolution:** `clientMutationId` (UUID) sent with every mutation. Server deduplicates on this field. Two partners logging offline for the same time window is fine — events have different IDs and are not conflicting (they're additive, not competing for the same slot).
- **Retry policy:** Exponential backoff: 1s, 2s, 4s, 8s, max 3 retries per mutation. After 3 failures: surface to user.
